title: "Send Your First Message with DRMQ: Java Quickstart Guide"
sidebarTitle: "Quickstart"
description: "Clone, build, and run DRMQ locally, then send and receive your first message end-to-end using the DRMQProducer and DRMQConsumer Java APIs."
---

This guide walks you through getting a DRMQ broker running on your machine and exchanging your first message end-to-end. You'll clone the repository, build all modules with Maven, start a single-node broker, and use `DRMQProducer` and `DRMQConsumer` to produce and consume a message. The whole process takes less than five minutes.

<Steps>

  <Step title="Check Prerequisites">
    Confirm you have the required tools installed before you begin.

    | Tool | Minimum version | Check command |
    |---|---|---|
    | Java (JDK) | 17 | `java -version` |
    | Apache Maven | 3.8.x | `mvn -version` |

    <Tip>
      If you don't have Maven installed locally, don't worry — the repository ships with a Maven wrapper (`mvnw`). You can substitute `./mvnw` for every `mvn` command in this guide.
    </Tip>
  </Step>

  <Step title="Clone and Build">
    Clone the repository and run a full build from the root directory. Maven will compile all modules, generate the Protocol Buffers Java classes from `drmq-protocol`, and package the broker and client JARs.

    ```bash
    git clone https://github.com/samuel025/DRMQ.git
    cd DRMQ
    mvn clean install
    ```

    A successful build ends with output similar to:

    ```text
    [INFO] BUILD SUCCESS
    [INFO] Total time: ...
    ```

    <Note>
      The first build downloads dependencies from Maven Central. Subsequent builds are significantly faster because artifacts are cached in your local `~/.m2` repository.
    </Note>
  </Step>

  <Step title="Start the Broker">
    Start a single-node broker on port `9092`. The `--data-dir` flag tells the broker where to persist its Write-Ahead Log segments.

    ```bash
    ./mvnw -pl drmq-broker exec:java -Dexec.args="--port 9092 --data-dir ./data-1"
    ```

    Watch the logs for a line confirming the broker is ready to accept connections:

    ```text
    INFO  c.d.b.BrokerServer - DRMQ Broker started on port 9092 with data directory ./data-1
    ```

    Leave this terminal open. Open a new terminal for the next steps.

    <Tip>
      To run a fault-tolerant three-node cluster instead, see the [Cluster Mode](/concepts/cluster-mode) guide.
    </Tip>
  </Step>

  <Step title="Send a Message">
    Add `drmq-client` as a dependency in your project's `pom.xml`, then use `DRMQProducer` to publish a message. The producer accepts a comma-separated list of bootstrap servers so it can automatically fail over if one node goes down.

    ```xml
    <dependency>
        <groupId>com.drmq</groupId>
        <artifactId>drmq-client</artifactId>
        <version>1.0.0-SNAPSHOT</version>
    </dependency>
    ```

    ```java
    import com.drmq.client.DRMQProducer;

    // Pass a comma-separated bootstrap list for automatic failover.
    try (DRMQProducer producer = new DRMQProducer("localhost:9092")) {
        producer.connect();

        DRMQProducer.SendResult result = producer.send("my-topic", "Hello, DRMQ!");

        if (result.isSuccess()) {
            System.out.println("Message sent at offset " + result.getOffset());
        } else {
            System.err.println("Send failed: " + result.getErrorMessage());
        }
    } catch (IOException e) {
        e.printStackTrace();
    }
    ```

    A successful send prints the assigned log offset, for example:

    ```text
    Message sent at offset 0
    ```

    <Note>
      `DRMQProducer` implements `AutoCloseable`. Wrapping it in a try-with-resources statement ensures the TCP connection is always closed cleanly.
    </Note>
  </Step>

  <Step title="Consume a Message">
    Use `DRMQConsumer` to read the message you just sent. Provide a consumer group name to enable load-balanced group mode — the broker coordinates offset dispatch so each message is delivered to exactly one consumer within the group.

    <Tabs>
      <Tab title="Group Mode (Recommended)">
        Group mode is the default when you supply a consumer group name. The broker tracks offsets for you and distributes messages across all consumers sharing the same group.

        ```java
        import com.drmq.client.DRMQConsumer;
        import java.io.IOException;
        import java.util.List;

        try (DRMQConsumer consumer = new DRMQConsumer("localhost:9092", "my-group")) {
            consumer.setAutoCommit(true);
            consumer.connect();
            consumer.subscribe("my-topic");

            while (true) {
                List<DRMQConsumer.ConsumedMessage> messages = consumer.poll(100, 1000);
                for (DRMQConsumer.ConsumedMessage msg : messages) {
                    System.out.printf("Received (offset %d): %s%n",
                            msg.offset(), msg.payloadAsString());
                }
            }
        } catch (IOException e) {
            e.printStackTrace();
        }
        ```
      </Tab>
      <Tab title="Single Mode (Manual Offset Control)">
        Disable group mode when you need precise control—for example, to replay messages from a specific offset.

        ```java
        import com.drmq.client.DRMQConsumer;
        import java.util.List;

        try (DRMQConsumer consumer = new DRMQConsumer("localhost:9092", "my-group")) {
            consumer.setGroupMode(false); // Disable broker-side coordination
            consumer.connect();

            // Start from offset 0 to replay from the beginning
            consumer.subscribe("my-topic", 0);

            while (true) {
                List<DRMQConsumer.ConsumedMessage> messages = consumer.poll(100, 1000);
                for (DRMQConsumer.ConsumedMessage msg : messages) {
                    System.out.printf("Replaying (offset %d): %s%n",
                            msg.offset(), msg.payloadAsString());
                }

                // Manually commit after processing
                if (!messages.isEmpty()) {
                    long lastOffset = messages.get(messages.size() - 1).offset();
                    consumer.commit("my-topic", lastOffset + 1);
                }
            }
        } catch (IOException e) {
            e.printStackTrace();
        }
        ```
      </Tab>
    </Tabs>

    <Note>
      `poll(maxMessages, timeoutMs)` uses long-polling: the broker waits up to `timeoutMs` milliseconds before returning an empty response. This keeps your polling loop efficient without busy-waiting.
    </Note>
  </Step>
